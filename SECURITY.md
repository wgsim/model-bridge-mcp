# Security Policy

## Supported Versions

Security fixes are applied to the latest state of `main`.

## Reporting a Vulnerability

Please do not open a public issue for suspected security vulnerabilities.

Report security issues privately through GitHub security reporting if it is available for this repository. If private reporting is not available, contact the maintainer through GitHub before sharing details publicly.

When reporting an issue, include:

- A short description of the vulnerability
- Affected files, commands, or configuration
- Reproduction steps
- Expected impact

You can expect an initial response within a reasonable maintenance window, but no fixed SLA is guaranteed.
